# Code of Conduct

DoneProof is built for calm, evidence-first collaboration.

Expected behavior:

- be direct and respectful;
- bring receipts when reporting bugs;
- avoid sharing secrets or private customer data;
- assume mistakes are fixable, then help fix them.

Unacceptable behavior:

- harassment;
- publishing private data;
- pressuring maintainers to add unsafe defaults;
- submitting examples with real secrets or private names.
