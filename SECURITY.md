# Security Policy

DoneProof is local-first and should never require secrets to validate a receipt.

## Do Not Share

Do not include the following in receipts, issues, examples, or public logs:

- API keys;
- tokens;
- `.env` contents;
- private customer names;
- private project names;
- internal machine paths;
- private bridge or runtime configuration.

## Reporting Security Issues

Use GitHub private vulnerability reporting:

```text
https://github.com/giugiu-a11y/doneproof/security/advisories/new
```
